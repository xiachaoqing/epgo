## ✅ 文章优化标准

### 内容质量标准

**【符号规范】**
- ✅ 句号：。（中文） / . / 。（英文）
- ✅ 问号：？（一个，不是？？）
- ✅ 感叹号：！（一个，不是！！）
- ✅ 无多余标点符号

**【格式规范】**
- ✅ 标题长度：10-30字
- ✅ 段落长度：300-500字为宜
- ✅ 字体大小：统一（14-16px）
- ✅ 行距：统一（1.5-1.8倍）
- ✅ 小标题：使用H2/H3结构

**【内容规范】**
- ✅ 字数标准：
  - 优质：≥2000字
  - 中等：1500-2000字
  - 需改进：<1500字
- ✅ 结构完整：
  - 开头：引入主题
  - 中间：详细讲解
  - 结尾：总结和建议
- ✅ 实用性：包含例句、技巧、练习等

## 📊 数据库结构

### 文章表（ep_news）

```sql
id              - 文章ID
title           - 文章标题
content         - 文章内容（HTML格式）
description     - 文章摘要
keywords        - SEO关键词
author          - 作者名
hits            - 阅读数
likes           - 点赞数
comments        - 评论数
create_time     - 创建时间
update_time     - 更新时间
recycle         - 回收站标记（0=正常 / 1=已删除）
thumb           - 封面图URL
column_id       - 栏目ID（分类）
status          - 文章状态
```

### 常用查询

```sql
-- 查看文章总数和质量
SELECT COUNT(*) as 总数,
       SUM(CASE WHEN LENGTH(content) >= 2000 THEN 1 ELSE 0 END) as 优质数,
       ROUND(AVG(LENGTH(content))) as 平均字数
FROM ep_news WHERE recycle=0;

-- 查看符号错误
SELECT COUNT(*) as 多余问号
FROM ep_news
WHERE content LIKE '%??%' AND recycle=0;

-- 查看阅读数分布
SELECT COUNT(*) as 总数,
       SUM(CASE WHEN hits < 10000 THEN 1 ELSE 0 END) as 低阅读,
       SUM(CASE WHEN hits >= 10000 AND hits < 50000 THEN 1 ELSE 0 END) as 中阅读,
       SUM(CASE WHEN hits >= 50000 THEN 1 ELSE 0 END) as 高阅读
FROM ep_news WHERE recycle=0;
```
