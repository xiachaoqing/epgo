## 二、MetInfo 模板机制（核心规则，必须遵守）

### 2.1 CSS 加载管线（4层）

```
Layer 1: public/web/css/basic.css          ← 框架底层，不可动
Layer 2: templates/epgo-education/cache/metinfo.css  ← 框架自动编译生成
Layer 3: templates/epgo-education/cache/{page}_{lang}.css  ← 页面级
Layer 4: <style>body{...}</style>           ← 主题色内联
```

**cache/metinfo.css 编译内容 = config.json里的插件CSS + static/metinfo.css（变量替换后）**

### 2.2 添加自定义CSS的唯一正确方式

在 `config.json` 的 `css` 数组**末尾**追加：

```json
"{$metui_temp}css/epgo-education.css"
```

`{$metui_temp}` = 模板目录 `templates/epgo-education/`

**❌ 禁止在 head.php 里手写 `<link>` 标签**

### 2.3 触发框架重新生成 cache

```bash
ssh epgo '
rm -f /www/wwwroot/go.xiachaoqing.com/templates/epgo-education/cache/metinfo.css
rm -f /www/wwwroot/go.xiachaoqing.com/templates/epgo-education/cache/metinfo.js
rm -rf /www/wwwroot/go.xiachaoqing.com/cache/*
mkdir -p /www/wwwroot/go.xiachaoqing.com/cache/{config,data}
chmod -R 777 /www/wwwroot/go.xiachaoqing.com/cache/
'
# 然后访问一次首页，框架自动重新编译 cache/metinfo.css
```

### 2.4 模板文件结构规范

每个页面 PHP 文件**必须**是：
```php
<?php defined('IN_MET') or exit('No permission'); ?>
<include file="head.php" />   ← 包含 <met_meta/>，输出完整 DOCTYPE+head+body
<!-- 页面内容 -->
<include file="foot.php" />   ← 包含 <met_foot/>，输出JS+关闭body/html
```

**❌ 禁止在模板里写 `<!DOCTYPE>` `<html>` `<head>` `</body>` `</html>`**

### 2.5 head.php 的作用

- 第一行 `<met_meta page="$met_page" />` 由框架自动输出：
  - 完整的 `<!DOCTYPE html><html><head>...</head><body>`
  - 注入 `basic.css`
  - 注入 `cache/metinfo.css`（含自定义CSS）
- 之后是导航栏 HTML

### 2.6 foot.php 的作用

- 页脚 HTML
- 最后一行 `<met_foot />` 输出：
  - `cache/metinfo.js`（框架JS）
  - 统计代码
  - `</body></html>`

### 2.7 导航布局原理

- Bootstrap 4 Alpha 风格（class命名如 `pull-xs-left`、`m-t-20`、`text-xs-center`）
- 实现用 float，不是 flexbox
- Logo：`.mblogo`（手机）和 `.pclogo`（PC），CSS媒体查询在 **1366px** 断点切换
- 两张图同时存在DOM，通过 `display:none` 切换

### 2.8 文章列表 ajax 机制

`news.php` 里：
```php
<include file='ajax/news'/>
```
- **首次渲染**：服务端 include，直接输出列表HTML
- **翻页**：JS 发 Ajax 请求 `ajax/news.php`，返回HTML片段 append 进列表

`ajax/news.php` 是**同构渲染**，既可服务端include，也可客户端Ajax请求。

### 2.9 Bootstrap 断点（自定义CSS必须对齐）

```
xs: < 480px
sm: ≥ 480px
md: ≥ 768px
lg: ≥ 992px
xl: ≥ 1200px
xxl: ≥ 1600px
Logo切换特殊断点: 1366px
```

### 2.10 CSS 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 间距 | `m/p-方向-数值` | `m-t-20`, `p-x-5` |
| 文字对齐 | `text-尺寸-方向` | `text-xs-center` |
| 浮动 | `pull-尺寸-方向` | `pull-md-right` |
| 隐藏 | `hidden-尺寸-方向` | `hidden-sm-down` |
| 自定义 | `epgo-` 前缀 | `epgo-hero`, `epgo-card` |

**❌ 自定义CSS 禁止覆盖 `met-`、`navbar`、`container`、`row`、`col-` 等框架class**

## 五、Nginx 路由配置

**文件**：`/www/server/panel/vhost/rewrite/go.xiachaoqing.com.conf`

```nginx
location / {
    try_files $uri $uri/ @metinfo;
}
location @metinfo {
    rewrite ^/(.*)$ /news/index.php last;
}
```

**说明**：MetInfo 的栏目页（`ket/`、`daily/` 等）需要路由到 `news/index.php`（`M_NAME=news`），不是根目录的 `index.php`（`M_NAME=index`）。MetInfo 通过 `REQUEST_URI` 中的 foldername 识别具体栏目。

## 八、已知问题与解决方案

| 问题 | 原因 | 解决 |
|------|------|------|
| CSS改了不生效 | cache/metinfo.css 未重新生成 | 删除cache文件，访问一次首页 |
| 栏目页显示首页内容 | Nginx路由到了根index.php(M_NAME=index) | rewrite到news/index.php |
| 文章列表无样式 | ajax/news.php的class名与CSS不一致 | 保持class名统一 |
| Logo两个都显示 | static/metinfo.css未被编译进cache | 删除cache让框架重新生成 |
| 导航栏跑到底部 | 自定义CSS覆盖了框架导航class | CSS只用epgo-前缀，不动met-/navbar- |
| 页面相对路径带../ | 路由经过news/index.php导致路径层级变化 | 正常现象，MetInfo内部处理 |
| 普通导航点击无效 | JS 误拦截了过宽的 `.dropdown a.nav-link` 选择器 | 只绑定 `.dropdown > a.nav-link[data-hover]`，普通栏目走浏览器默认跳转 |
| KET/PET 子菜单不显示 | 不能依赖 `$m['sub']` / `tag action='category' type='son'` 在当前模板上下文自动正确输出 | 在 `head.php` 里直接查 MySQL 的 `ep_column.bigclass` 动态构建子菜单 |
| 再次误改导航风险 | 只看模板变量容易误判，真实问题可能在最终渲染 HTML 或 JS 事件 | 先检查线上 HTML，再检查 JS click handler，最后再改模板 |

### 8.1 导航修复规则（本项目必须遵守）

1. `head.php` 中普通栏目链接必须保留原生 `<a href="...">` 跳转行为。
2. 下拉菜单父项（KET/PET）才允许 JS 接管点击。
3. `metinfo.js` 中导航点击绑定只能使用：

```js
var aLink = $(".met-nav").find('.dropdown > a.nav-link[data-hover]');
```

**禁止**改回以下写法：

```js
$(".met-nav").find('.dropdown a.nav-link')
```

因为这会把普通栏目也错误卷进去，导致“英语阅读 / 每日英语 / 资料下载 / 关于我们”点击异常。

4. 每次改导航后，必须做这 4 步：
   - 同步服务器
   - 清 `cache/`
   - 强刷浏览器缓存
   - 核对线上最终 HTML 是否正确

5. 判断问题归因顺序：
   - 先看线上 HTML 的 `href` 是否正确
   - 再看是否有 JS click handler 拦截
   - 最后才看模板变量是否有问题

### 8.2 封面图规则（必须遵守）

1. **禁止**用历史站点旧图（如 `/upload/202403/*.jpg`）充当当前英语教育站点栏目默认封面。
2. 当前站点的默认封面只能来自以下两类：
   - 临时占位：`/upload/epgo-covers/*.png`
   - 正式新图：`/upload/epgo-photo-covers/{ket|pet|reading|speech|daily|download|about}/...`
3. 如果要替换栏目默认封面，必须先把新图放进 `upload/epgo-photo-covers/` 新目录，再批量更新数据库。
4. **不要**为了“先看起来正常”而复用其他旧项目图片，这会造成场景错误，比默认图更糟。
5. 后续如果要做批量封面更新，优先策略应为：
   - 新图落本地
   - 只替换当前仍使用默认占位图的文章
   - 不覆盖已有明确业务图
