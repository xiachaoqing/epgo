### ✅ 5. 二级栏目访问问题根本解决

**问题分析**:
- 用户点击二级菜单（KET真题等）返回404
- 二级栏目有foldername（ket-exam、ket-word等），但框架无法通过foldername找到它们

**根本原因**:
MetInfo框架的`get_column_folder()`方法(column_label.class.php:270) 有如下条件:
```php
if ($val['foldername'] == $floder && ($val['classtype'] == 1 || $val['releclass'])) {
    return $val;
}
```
即：**只有classtype==1或releclass有值的栏目才能通过foldername访问**

**框架设计理解**:
- 一级栏目（classtype=1）: 独立访问 `/ket/`，显示该栏目下的所有文章
- 二级栏目（classtype=2）: 不应独立访问，只在一级栏目页面中通过 subcolumn_nav.php显示

**解决方案**:
```sql
-- 清除二级栏目的foldername，防止用户误访问
UPDATE ep_column SET foldername='' WHERE lang='cn' AND classtype=2;
```

**修复后的行为**:
- `/ket/` → 显示KET备考的所有文章 + 二级菜单栏（真题、词汇、写作、听力）
- 点击二级菜单 → 调用 subcolumn_nav.php，通过 `type='son'` 和 `cid=101` 过滤该一级栏目下的指定二级栏目
- 二级栏目无法直接访问（这是框架正确的设计）

## 关键学习点

### MetInfo框架栏目体系
1. **一级栏目** (classtype=1)
   - 有独立的foldername
   - 可直接访问 `/foldername/`
   - 显示该栏目的所有文章列表
   - 可有子栏目

2. **二级栏目** (classtype=2)
   - 通常不需要foldername
   - 不能独立访问
   - 在一级栏目的subcolumn_nav.php中显示
   - 通过参数过滤显示

3. **URL路由流程**
   ```
   /ket/
   → REQUEST_URI解析: foldername = 'ket'
   → get_column_folder('ket')
   → 查找 foldername='ket' AND classtype=1
   → 找到id=101(KET备考)
   → 加载news.php模板
   → news.php包含subcolumn_nav.php
   → subcolumn_nav.php显示该一级栏目的所有子栏目
   ```

### 模板加载顺序
```
head.php
  ↓ 导航、banner
news.php （一级栏目或文章列表）
  ↓
subcolumn_nav.php （二级菜单栏，只在classtype=2栏目时包含）
  ↓
文章列表内容
  ↓
foot.php
```
