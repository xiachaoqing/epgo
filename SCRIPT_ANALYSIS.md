## 📊 epgo 脚本运行架构分析

### 当前定期运行的任务

**1️⃣ 每日 08:00 - daily_generate_articles_epgo.sh**
- 运行：每天早上8点
- 功能：每天插入1-2篇随机文章（按7天周期轮转题目）
- 覆盖：103-105, 111-114, 121-124 栏目
- 输出日志：`/var/log/epgo_article_gen.log`

**2️⃣ 每日 03:15 - daily_update_epgo.py**
- 运行：每天凌晨3:15分
- 功能：针对每个栏目插入主题文章（轮转话题）
- 覆盖：103-105, 111-114, 121-124 栏目
- 输出日志：`/www/wwwroot/go.xiachaoqing.com/cache/daily_update_epgo.log`

### ⚠️ 发现的问题

1. **重复插入风险**
   - 两个脚本都在往数据库插文章
   - 没有去重逻辑，可能导致相同文章被插入多次
   - 建议：根据 title + class1 组合检查是否已存在

2. **时间随机性不足**
   - daily_generate_articles_epgo.sh 固定为 NOW()
   - daily_update_epgo.py 也固定为当前时间
   - 结果：一天内插入的所有文章 updatetime 都几乎相同
   - 建议：随机化时间戳（过去7-30天内）

3. **无效文章生成**
   - daily_generate_articles_epgo.sh 中 SQL 未转义处理有限
   - content 包含 HTML，可能导致字符转义问题

4. **缺少数据质量检查**
   - 没有验证插入是否成功
   - 没有检查重复数据
   - 没有定期清理垃圾数据

5. **栏目 106、107 被遗漏**
   - 两个脚本都没有为 Download 和 About 栏目添加内容
   - 结果：这两个栏目永不更新

6. **无缓存清理机制**
   - MetInfo 生成的静态 HTML 可能过期
   - 建议：新文章插入后自动清理相关缓存

### 💡 优化建议

【优先级高】
- [ ] 合并两个脚本，避免重复
- [ ] 为 106、107 栏目补充定期更新
- [ ] 随机化文章时间戳（让数据看起来更真实）
- [ ] 增加数据去重检查

【优先级中】
- [ ] 改进 SQL 字符转义
- [ ] 添加缓存自动清理
- [ ] 添加运行成功/失败通知

【优先级低】
- [ ] 统计每月增长情况
- [ ] 添加数据质量监控
- [ ] 记录详细的执行日志

### 📋 建议的改进方案

```
功能：

4. 验证插入成功
```
