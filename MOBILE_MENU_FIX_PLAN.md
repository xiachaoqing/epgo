# 🎯 手机版菜单下拉颜色优化计划

**问题描述**: 手机版本下拉菜单中的某些文字颜色太深，与蓝色背景对比度不足
**问题位置**: `foot.php` 中的汉堡菜单
**优先级**: 🔴 高优先级
**修复时间**: 立即执行

---

## 📋 问题分析

### 当前问题
```
手机菜单显示的文字颜色:
- "KET真题解析" ❌ 颜色太深，看不清
- "KET词汇速记" ❌ 颜色太深，看不清
- "KET写作指导" ❌ 颜色太深，看不清
- "KET听力技巧" ❌ 颜色太深，看不清

根本原因:
1. 文字颜色是深蓝色 (#1e40af 或类似)
2. 菜单背景是蓝色 (#1e40af 或类似)
3. 对比度不足，导致看不清
```

### 解决方案
```
方案A: 改变文字颜色为白色或浅色
方案B: 改变菜单背景为深色
方案C: 同时优化文字和背景颜色

我们选择: 方案A (改变文字颜色)
原因: 最直接、最简单、效果最好
```

---

## 🔧 修复方案

### 修复位置
```
文件: templates/epgo-education/foot.php
位置: 汉堡菜单部分 (met-menu-list)
```

### 修复方法

**步骤1**: 找到菜单文字颜色定义
```css
当前: color: #1e40af (或 #6b7280)
修改为: color: #ffffff (白色)
        或 color: rgba(255,255,255,0.9) (半透明白色)
```

**步骤2**: 确保悬停时有视觉反馈
```css
正常: color: rgba(255,255,255,0.8)
悬停: color: #ffffff
背景: background: rgba(255,255,255,0.1)
```
